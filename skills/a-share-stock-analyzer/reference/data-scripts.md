# 数据脚本字段口径明细

> SKILL.md【数据获取】章节的**展开说明**。SKILL.md 只留紧凑表 + 铁律 + 交叉验证流程；
> 每个脚本的字段口径、CLI 变体、边界情形（非两融/新股/限流退化）都在本文件。
> 所有"当日数字"一律走脚本，**不采信 WebSearch 摘要**（铁律见 SKILL.md）。
> 脚本不可用时手抓接口见 `manual-data-interfaces.md`。

---

## `analyze.py` — 编排器（新建分析优先入口）

并发跑齐 `quote`+`funds`+`valuation`+`news`+`events`（未来事件日历）+`breadth`（市场宽度/情绪），附带大盘指数（并算**个股相对强弱 RS** 与**大盘均线择时**），**定位个股所属行业板块、给出板块当日涨跌/板块主力净流入与「个股 vs 板块」相对强弱**（确定性判断"逆板块独跌"还是"随板块杀跌"），最后产出**研判脚手架**：按行业自动建议 Step 3.1 权重模板、预填 `score.py` 命令（含估值护栏分位、ST 封顶标志）、列出仍需 WebSearch 的维度。

```bash
python3 scripts/analyze.py 600519              # 行情+资金+估值+大盘+脚手架, 一次到位
python3 scripts/analyze.py 600519 --no-index   # 不附带大盘指数
python3 scripts/analyze.py 600519 --lhb-days 30 # 自定义龙虎榜回看
python3 scripts/analyze.py --json 600519        # JSON, 供自动化消费
python3 scripts/analyze.py 600519 --cache 60    # 开 60 秒磁盘缓存(/loop 或连分析多只时)
```

- 东财请求统一限并发 + **指数退避抖动重试**（push2 退避基数更长）+ **并发同 URL 去重（in-flight coalescing，6 个子脚本撞同一指数/板块只打一次）**，缓解 push2/datacenter 限流；板块 K线为 best-effort，限流时仍给当日板块 RS。
- 编排器只是入口+脚手架，**不替代子脚本**（下文逐一说明口径），也**不自动算分**（四维方向/强度是模型判断，国际/经济需搜索，故只预填 `score.py` 命令）。
- 退出码：`0` 至少一路数据可用；`2` 全部失败（网络/限流）。

## `quote.py` — 行情 / 主力资金 / 技术锚点

脚本已把"双源抓取 + 时间戳校验 + 交叉验证 + 主力资金口径 + 价格缩放"固化，**优先用它，不要手搓 URL**。

```bash
python3 scripts/quote.py 600519                      # 个股, 自动判市场
python3 scripts/quote.py 600519 sh000001 sh000300    # 个股+大盘一次拉全
python3 scripts/quote.py --json 600519               # JSON
```

输出每个标的：现价、涨跌幅、换手率、**今日主力净流入 + 近5日累计**（东财权威口径）、**技术锚点（MA20/MA60、近20/60日前高前低、换手率20日均值与历史分位）**、所属行业、腾讯/东财两源时间戳。

数据源：腾讯 `qt.gtimg.cn`（纯文本、秒级时间戳）+ 东财 `push2/push2his.eastmoney.com`（纯 JSON）。两路独立、不依赖 JS、各自带时间戳，互为交叉验证。

退出码（供 `/loop` 等判可信度）：`0`=至少一标的可直接写入（一致/单源且非滞后）；`3`=抓到数据但无任何可信标的（全部打架/滞后），需人工核实勿采用；`2`=全部抓取失败（网络/限流）。

## `valuation.py` — 估值快照 + 历史分位 + 护栏

```bash
python3 scripts/valuation.py 600519              # 单只
python3 scripts/valuation.py 600519 000858       # 多只
python3 scripts/valuation.py --json 600519        # JSON
```

- 输出东财权威口径的 PE-TTM、PB、动态/静态 PE、**市销率 PS-TTM、PEG、股息率（TTM 现价口径）**、总市值、流通市值及时间戳；亏损股标 `亏损(PE<0)`，此时看 **PB / 市销率**。
- **PE-TTM / PB / PS 历史分位**由脚本接口直算（东财约8年日序列本地算近3年、近5年分位，口径=当前值高于历史多少比例的交易日，分位越高越贵）。脚本直接给**估值护栏建议**（分位>90%下调一档 / <10%上调一档 / 中间不调），**不走 WebSearch**。PE<0（亏损）日不计入 PE 分位，护栏自动退到 PB 分位。
- **股息率**=接口直算 TTM 现价口径（近12个月已除权每股税前派现 / 现价），"近12月无派现"会明确标注；其**历史分位也由脚本接口直算**（用日序列每个交易日的收盘价+往前12月派现重建逐日股息率序列算近3/5年分位，**方向与估值相反：分位越高=派息相对历史越慷慨/估值越便宜**；近似口径，未做送转复权），**不走 WebSearch**。
- 估值仅作**护栏**（极端高/低估时综合分 ±1 档，见 Step 3.3），不单独决定方向。
- **限流注**：东财快照接口（push2）偶发限流时，脚本仍照常输出 datacenter 来源的分位/PS/股息（独立通道），只是 PE/PB 当前值与市值暂缺——勿因快照失败误判全部缺数。

## `funds.py` — 融资融券 + 龙虎榜

资金维度里的**融资融券余额、龙虎榜机构/游资席位**是"硬资金事实"，**不采信 WebSearch 摘要**（摘要常把上月余额、隔日龙虎榜当成今日）。走东财 datacenter 权威接口：

```bash
python3 scripts/funds.py 600519              # 融资融券近5日趋势 + 龙虎榜近90天
python3 scripts/funds.py 600519 000858       # 多只
python3 scripts/funds.py --json 600519        # JSON
python3 scripts/funds.py --lhb-days 30 600519 # 自定义龙虎榜回看窗
```

- **融资融券**：输出融资余额、占流通市值%、当日融资买入额、融券余额，及近5日融资余额**趋势（回升/下降）**。东财盘后披露、通常滞后约1个交易日，脚本用与 quote.py 一致的 `freshness` 标注口径（`📅 最近交易日收盘口径` 可用须标注；`⛔ 滞后` 勿用）。**非两融标的**会明确返回"非两融标的"，不是报错。
- **龙虎榜**：输出近期每次上榜的日期、上榜原因、龙虎榜净买额、机构概述（如"5家机构买入"）、当日涨跌幅。脚本对最近一次上榜判新鲜度：**仅"今日/最近交易日"上榜计入当日资金方向**，更早的标"历史背景, 不计入当日方向"。**多数股票多数日子不上榜，"近N天无上榜记录"是正常状态，如实写"无"即可。**
- 退出码：`0`=至少一接口抓到；`2`=全部失败（网络/限流，datacenter 偶发限流属正常，勿当代码 bug）。

## `news.py` — 近期公告（事件背景）

近期公告的**标题 + 发布日期 + 栏目分类**是结构化硬事实（业绩预告/分红/股权激励/回购/监管问询/停复牌等），**不采信 WebSearch 摘要**（摘要常把旧公告/传闻当成今日）。走东财公告中心接口：

```bash
python3 scripts/news.py 600519              # 近14天公告
python3 scripts/news.py --days 30 600519      # 自定义回看天数
python3 scripts/news.py --json 600519         # JSON
```

- 只给"标题+日期+栏目"，不抓正文、不做解读——**解读是模型的活，脚本只供事实**。
- 公告是"已发生事件"，**不计入当日资金/行情打分**；作事件背景与情绪/基本面佐证。
- 多数日子无公告很正常，空结果如实写"近 N 天无公告"。`analyze.py` 已并发带上本路。

## `events.py` — 未来事件日历（解禁/分红除权/业绩预告）

`news.py` 只抓**已发生**公告；但A股最强的可预期催化剂是**未来已知日程**——这些同样是结构化硬事实，**不采信 WebSearch 摘要**（摘要常把旧解禁/旧预案/旧预告当成最新）。走东财 datacenter：

```bash
python3 scripts/events.py 600519              # 解禁/分红除权/业绩预告
python3 scripts/events.py --json 600519        # JSON
python3 scripts/events.py --lookback-days 365 600519   # 已发生事件回看窗
```

- **限售解禁**（`RPT_LIFT_STAGE`）：未来解禁日 + 解禁股数 + **占流通比** + 解禁市值 + 限售类型（首发/定增/股权激励）。大比例解禁 = **确定性供给冲击**，临近（默认60天内、占流通≥3%）会进"临近/重点事件"提示——纳入**维度一资金面**（抛压）与 **Step 1.5 体检**。
- **分红除权**（`RPT_SHAREBONUS_DET`）：最新方案进度（预案/股东大会通过/实施分配）+ 每10股税前派现 + 未来**股权登记日/除权除息日**。登记日临近（默认30天内）影响短期情绪（填权/贴权预期）。
- **业绩预告**（`RPT_PUBLIC_OP_NEWPREDICT`）：类型（预增/预减/扭亏/首亏）+ **净利同比变动幅度** + 报告期。是财报披露前最硬的**基本面方向**（维度四）。**带时效门**：公告超120天视为"可能已实现"，只作背景、不当前瞻信号高亮（脚本已自动判 `forward`）。
- 多数票多数时候"无临近事件"是正常状态，空即如实写"无"。`analyze.py` 已并发带上本路，并把"临近/重点事件"提炼进研判脚手架。

## `breadth.py` — 市场宽度 / 情绪温度（全市场）

情绪是**全市场层面**的——两市成交额与涨停/跌停家数是比"大盘涨跌幅"更直接的情绪温度计，**不采信 WebSearch 摘要**。走纯数据接口：

```bash
python3 scripts/breadth.py              # 成交额+涨跌停家数+主要指数
python3 scripts/breadth.py --json        # JSON
```

- **两市成交额** = 上证综指（全沪）+ 深证综指（全深）的 `f6` 之和（`ulist.np` 实时；优于"上证+深成"，后者深成仅500只不全）= 流动性/风险偏好。
- **涨停/跌停家数**（push2ex 涨停池/跌停池 `data.tc`，按真实交易日历取交易日）= 打板情绪 vs 退潮恐慌；脚本据此给"情绪温度"分档（🔥偏暖/❄️恐慌/🌡️中性，era-robust 不依赖成交额绝对值）。
- 主要指数（上证/深成/创业板）涨跌幅作风险偏好背景。盘中为滚动值、非最终收盘。
- ⚠️ **不含北向**：个股与市场聚合的北向实时数据均已停披，禁止臆造。
- `analyze.py` 附带大盘指数时已并发带上本路（`--no-index` 时跳过）。

## `position.py` — 持仓管理（用户已有仓位时）

把 Step 6「操作参考」从静态文字升级为**确定性算术**：用户给出持仓股数+成本，算盈亏平衡、各价位 PnL 阶梯、止损/目标（挂靠真实技术位）、盈亏比 R:R，并评估加/减仓情景。**别手算 PnL 表**（口径易错），算术交脚本；模型只结合方向研判给"减/加/守/止损"判断。

```bash
python3 scripts/position.py 000066 --shares 3500 --cost 18.2                 # 基础: 平衡价+PnL阶梯+止损/目标+R:R
python3 scripts/position.py 000066 --shares 3500 --cost 18.2 --stop 17.5 --target 20.78   # 手填止损/目标
python3 scripts/position.py 000066 --shares 3500 --cost 18.2 --add-shares 1000 --add-price 17.7   # 加仓→新均价/新平衡
python3 scripts/position.py 000066 --shares 3500 --cost 18.2 --trim-shares 1500 --trim-price 18.4 # 减仓→已实现盈亏+剩余
python3 scripts/position.py 000066 --shares 3500 --cost 18.2 --price 17.8 --no-fetch   # 限流/手填现价(无K线则止损目标须手填)
```

- **现价 + 技术锚点**（MA20/MA60/近20/60日前高前低）走 `quote.analyze_one`（接口权威，不搜摘要），带新鲜度/交叉验证标注。
- **盈亏平衡**：成本价 + 含费近似（`--fee-pct` 默认 0.13% 双边，佣金+印花估算）。
- **止损/目标自动挂靠**：止损=现价下方最近支撑、目标=上方最近阻力；**带 1.5% 缓冲**——锚点紧贴现价时会跳过（否则止损被噪声秒扫、R:R 失真），都太近则退最近的并标 `⚠️距现价过近`。**无 K线（新股/停牌）则留空、要求 `--stop/--target` 手填，不臆造**（同 SKILL Step 6 铁律）。
- **盈亏比 R:R** = 现价上方空间 ÷ 下方风险；风险分母过小（<1% 或挂靠过近）时标 `⚠️失真`，提示换更下方支撑再看。
- **加仓**：新股数/新均价/新含费平衡价/加仓后浮盈亏。**减仓**：已实现盈亏 + 剩余股数(均价不变)/剩余现值。
- 退出码：0 正常；2 现价抓取失败且未 `--price` 手填（无法算 PnL）。`--json` 供自动化。
- ⚠️ 只做算术与价位挂靠，**不替代方向研判**（方向仍看 `analyze`/`score`）。

## 运维工具（不在主流程）

### `selftest.py` — 接口契约自检

报告数字反常、某维总是空、或怀疑东财改了字段时，先跑 `python3 scripts/selftest.py`——对固定活跃股逐接口断言关键字段，输出每项 PASS/FAIL/THROTTLED：**FAIL=接口字段/格式真变了（要对照 `manual-data-interfaces.md` 改 fetch_\*）**，THROTTLED=只是限流（稍后自愈，非 bug）。退出码 1=有契约不符、2=全限流无法判定。**首行附「交易日历有效期」体检（🗓️ WARN）**：节假日表临近/已过期（`trading_calendar.py` 的 `CALENDAR_VALID_THROUGH`）会提前告警，提醒补次年节假日，免得 freshness 静默退回启发式、长假误判；WARN 不影响退出码。

### `tests/` — 离线单元测试

确定性内核（`score` 算术/归一化/护栏/ST封顶、`review` 命中判定、`quote` 分位·收益·均线排列、`trading_calendar`）有纯函数测试，无网络。改这些脚本后跑 `python3 -m unittest discover -s tests`（秒级），防止口径静默漂移。

### `--cache` — 减少重复请求

`/loop` 反复跑或一次连分析多只时，`analyze.py --cache 60`（或环境变量 `ASHARE_CACHE_TTL=60`）开 60 秒磁盘缓存，TTL 窗口内同一接口直接读盘、不再打东财，缓解自我限流。**默认关闭**；因新鲜度始终按数据自带时间戳判定，命中旧缓存也会如实标 `last_close/stale`，**不破坏实时铁律**；实时盘中精读建议关或用很短 TTL。
