# A股个股深度分析

基于资金流量、市场情绪、国际形势、经济形势四个维度，对A股特定个股进行综合分析，给出走势研判与操作建议。

**核心原则：所有数据精确到小时级——以用户发指令的时间为基准。搜索关键词必须含完整日期（`YYYY年MM月DD日`）；行情/资金数据超过1个交易日须标注滞后，超过窗口的旧数据一律不计入信号打分（见【执行注意事项】）。**

---

## 【数据获取】当日行情与资金流——唯一权威章节

> 本节是所有"今日数字"的唯一来源规范。Step 2 各维度只引用本节，不再重复。

### ⚠️ 一条铁律

**今日股价 / 涨跌幅 / 主力资金 / 大盘指数 / 板块涨跌 / 换手率，一律不得采信 WebSearch 的 AI 摘要。** 搜索摘要常返回数周前的滞后价、"--" 占位符，或把不同日期/不同标的的数字错位拼接（实战翻车：曾把摘要里"上证+0.12%"写进报告，接口核实当日实际是 **-1.70%**，方向完全相反，连带情绪研判全错）。这类当日盘面事实必须用下面的**纯数据接口**抓取并核验时间戳。

### 脚本总览（字段口径明细见 `reference/data-scripts.md`）

**新建分析优先用编排器 `analyze.py`**：一条命令并发跑齐下列全部子脚本 + 大盘指数（个股相对强弱 RS / 均线择时）+ 板块定位（板块涨跌/主力净流入/「个股 vs 板块」RS，确定性判断"逆板块独跌"还是"随板块杀跌"）+ 未来事件临近提示，并产出**研判脚手架**（按行业建议 Step 3.1 权重模板、预填 `score.py` 命令、列出仍需 WebSearch 的维度）。东财请求已统一限并发 + 退避抖动重试 + 同 URL 去重，缓解 push2/datacenter 限流。

```bash
python3 scripts/analyze.py 600519              # 行情+资金+估值+大盘+脚手架, 一次到位
python3 scripts/analyze.py --json 600519       # JSON, 供自动化消费
python3 scripts/analyze.py 600519 --no-index   # 不附带大盘指数
```

| 脚本 | 用途 | 退出码 |
|------|------|--------|
| `analyze.py` | 编排器：一次跑齐下列全部 + 大盘/板块 RS + 脚手架（**不自动算分**） | 0 有数据 / 2 全失败 |
| `quote.py` | 行情·涨跌幅·换手率·今日及近5日主力净流入·技术锚点(MA20/60·前高低·换手分位)·所属行业 | 0 可写 / 3 抓到但不可信 / 2 全失败 |
| `valuation.py` | PE/PB/PS/PEG/股息率当前值 + 近3/5年历史分位 + 估值护栏建议 | — |
| `funds.py` | 融资融券余额趋势 + 龙虎榜机构/游资席位 | 0 至少一接口 / 2 全失败 |
| `news.py` | 近期公告：标题+日期+栏目（已发生事件背景，不计入当日打分） | — |
| `events.py` | 未来事件日历：解禁/分红除权/业绩预告（前瞻催化剂） | — |
| `breadth.py` | 全市场宽度/情绪：两市成交额 + 涨停/跌停家数 + 主要指数 | — |
| `score.py` | Step 3 综合评分算术（见 Step 3） | — |
| `position.py` | 持仓管理（用户已有仓位时）：吃进 `--shares`/`--cost`，算盈亏平衡·各价位 PnL 阶梯·止损/目标(挂靠真实技术位)·盈亏比 R:R·加/减仓情景（见 Step 6） | — |
| `record.py`·`review.py` | 研判留痕 + 复盘命中率（见 Step 5） | — |

> 子脚本仍可单独调用，编排器**不替代**它们、也**不自动算分**（四维方向/强度是模型判断，国际/经济需搜索）。每个脚本的 CLI 变体、字段口径、边界情形（非两融/新股/限流退化）见 **`reference/data-scripts.md`**。⚠️ 个股北向/陆股通实时数据已于 2024-08-18 停披，禁止臆造（详见维度一）。

### quote.py 输出的两个正交维度（决定能否写入报告，务必记住）

`quote.py`（及 `analyze.py` 内含）对每个标的结论分**两个正交维度**，`funds.py`/`valuation.py` 等复用同一 `freshness` 口径：

**一致性**（两源价格是否互相印证）：
- `✅ 已交叉验证` → 可直接作为事实写入报告
- `⚠️ 仅单一来源` → 写入时须标注"未交叉验证"
- `❌ 两源不一致` → **不得写入报告**，按提示人工核实

**新鲜度**（已接入真实沪深节假日表 `trading_calendar.py`，长假/盘前不误判）：
- 无标注 → 今日数据（盘中或今日收盘）
- `📅 最近交易日收盘口径` → 今日是**非交易日（周末/节假日）**、或交易日**盘前（<09:30）尚无当日数据**用昨收，**可用但报告须标注此口径**、不得写成"今日"
- `⛔ 数据滞后, 勿用` → 交易日盘中/盘后却只拿到更早数据，**不得写入报告**
- 节假日表权威覆盖至 2026 年底；之后自动回退旧"周末+≤4日"启发式。新一年需把节假日补进 `trading_calendar.py` 并把 `CALENDAR_VALID_THROUGH` 推到年底。

### 备用路径：脚本不可用时手动抓接口

正常一律用脚本。仅当环境**无法运行脚本**时才手搓接口——完整接口表、字段口径、缩放约定见 **`reference/manual-data-interfaces.md`**（含行情/资金/K线/两融/龙虎榜/估值/股息/公告全部接口）。

### 时间戳与交叉验证（每条当日数据都要走）

1. **抓取**：用上面接口，不用 WebSearch 摘要。
2. **验时间戳（已感知交易日历）**：交易日确认来源"更新时间/数据日期" = 今日（盘中到 HH:mm 级），对不上今日即真滞后、直接弃用（曾出现静默返回 2021 年旧数据）；**非交易日（周末/节假日）**则最近一个交易日收盘即合法最新，脚本标 `📅 最近交易日收盘口径`，报告须注明该口径、不得写成"今日"。
3. **交叉验证**：今日收盘价/涨跌幅需 ≥2 个独立来源一致方可作"事实"；仅单一来源时数据后标注"⚠️ 仅单一来源，未交叉验证"。
4. **标注口径**：盘中写"盘中 HH:mm 口径，非最终收盘"；收盘后写"收盘口径"。报告顶部"分析时间"与数据口径一致。
5. **拿不到就如实写**：缺当日数据时写"⚠️ 数据未获取，请在东方财富/同花顺核实"，**不得猜测、不得用旧值顶替、不得臆断"普涨/普跌/逆势"**——"逆势独跌"还是"随板块杀跌"会直接改变情绪结论。

## 整体流程

1. **明确标的** — 确认股票代码/名称及关注角度
2. **四维并行调研** — 同时调研资金流量、市场情绪、国际形势、经济形势
3. **综合研判** — 交叉验证四维信号，判断多空力量对比
4. **输出结论** — 走势研判、关键价位、操作建议、风险提示

---

## Step 1：明确分析标的

用户仅给股票名称未给代码时，先搜索确认代码避免同名混淆。未说明关注视角时按上下文判断，默认覆盖短中期（1周至3个月）。**无需逐条追问，直接进入分析。**

---

## Step 1.5：前置状态体检（先于四维，命中即先报状态）

用**已有信号**（`quote.py` 返回的名称/涨跌幅、`fflow.reason`、`kline.bars`，必要时一条搜索）做一次体检；命中下表任一状态，**优先于四维结论**——先讲清状态与风险，再决定是否继续打分：

| 状态 | 识别方式（现成信号） | 闸门动作 |
|------|---------------------|---------|
| **ST / *ST / 退市风险** | `quote.py` 返回的**名称含 ST/\*ST/退** | 强制风险提示；仓位封顶"轻仓试探/回避"；综合分**看多侧封顶为"中性偏多"**，不得给"偏多/强烈看多" |
| **停牌** | 现价=昨收且成交为 0，或搜索确认停牌 | **跳过四维当日打分**；直接给"停牌中，复牌前观望"，只做基本面/事件定性 |
| **新股/次新**（上市约<60交易日） | `kline.bars < 60` 或 `fflow.reason=新股` | 技术位（MA/前高低/换手分位）**留空标注不可用**；估值分位不可用；提示"次新筹码不稳、高波动" |
| **一字板/连续涨跌停** | 涨跌幅≈±10%（科创/创业±20%）且换手极低 | 提示流动性风险，操作区间标注"难成交"，止损/目标按封板方向谨慎给 |
| **临近大比例解禁 / 临近除权** | `events.py`（`analyze.py` 已带）"临近/重点事件"提示 | 大比例解禁（占流通≥3%、60天内）= 供给抛压，**资金维度看多侧降温、风险提示**；除权登记日临近作情绪提示 |

> 体检无异常时一句带过（"标的状态正常，进入四维分析"），不展开。

## Step 2：四维并行调研

> **时效性要求**：先确认今日日期并写下 `当前日期 = YYYY年MM月DD日`，所有搜索词替换此占位符。搜索关键词必须含完整日期（如 `2026年5月11日`），盘中查询额外加 `今日`/`盘中`/`实时`；不得仅用年月（会返回数周前结果）。
>
> **当日行情/资金一律走【数据获取】章节**（脚本或接口）；WebSearch 仅用于季报、政策、宏观、国际、新闻等非当日数据，且每条都要核验发布日期。

### 维度一：资金流量分析

**目标**：判断主力资金净流入/流出，以及机构、游资动向。

- **今日/近5日主力净流入**：用【数据获取】拿东财 fflow 口径（`quote.py` 已直接给出）。
- **所属行业板块主力净流入**：`analyze.py` 一并给出（板块层面的资金方向）——个股主力净流入要与板块资金方向对照：个股净流入而板块净流出=个股被单独吸筹（强）；个股净流出而板块也净流出=随板块退潮（系统性）。
- **融资融券 + 龙虎榜**：走 `scripts/funds.py`（接口权威口径，**不搜摘要**）——给融资余额趋势、龙虎榜机构/游资净买额。字段口径详见 `reference/data-scripts.md`。
- 仅以下"接口拿不到"的项用 WebSearch 补充（含日期占位符，每条核验发布日期）：
  ```
  [股票名称/代码] 机构持仓 最新季报 陆股通持股 [当前日期]
  [股票名称/代码] 大单 异动 [当前日期]
  ```

> **⚠️ 北向/陆股通个股实时数据已停披（2024-08-18 起）**：交易所不再披露沪深股通的**个股**实时资金流向，市面"今日北向增持 XX 万股"的摘要多为滞后值或臆造拼接，**禁止当作今日信号写入**。只可引用**季报前十大股东里的陆股通持股**（标明报告期），作背景项、不计入当日方向打分。

重点指标：主力近5/20日累计方向、融资余额增减、龙虎榜机构/游资席位、机构季报持仓变化、陆股通季报持股变化（背景）、大单异动。

**信号解读**：
| 信号 | 含义 |
|------|------|
| 主力连续5日净流入 + 融资余额回升 | 强势做多 |
| 主力净流出 + 融资下降 + 龙虎榜机构净卖 | 多方减弱 |
| 大单买入但价格未涨 | 可能有压力位或对倒 |
| 机构季报大幅加仓 / 陆股通季报增持 | 中长期看好 |

### 维度二：市场情绪分析

**目标**：评估市场对该股的情绪温度和短期交易特征。

- **今日换手率/大盘指数/板块涨跌/是否逆势**：用【数据获取】（`analyze.py` 一次带个股+上证+沪深300+**所属行业板块**）。
- **换手率历史分位**：用【数据获取】脚本输出的"20日均值 / ≈均值N倍 / 历史分位%"，**不要凭感觉估**——情绪温度计的"过热/冰冷"以此为锚（如分位>90%或>均值2倍才算过热）。
- **相对强弱 RS（个股 vs 大盘，确定性）**：`analyze.py` 直接输出个股近 1/5/20 日收益减基准（沪深300/上证）收益。RS>0=跑赢大盘/相对强势（可能逆势走强），RS<0=跑输/相对弱势。**"是否逆势"以此为锚，不再凭目测**。
- **个股 vs 板块 RS + 板块资金流（确定性）**：`analyze.py` 定位个股所属行业板块，给出**板块当日涨跌、板块主力净流入**及**个股相对板块 RS**（个股涨跌 − 板块涨跌；板块 K线可用时附 5/20 日）。这是区分"**逆板块独跌**"（个股跌而板块涨/平 → 个股自身利空，强信号）与"**随板块杀跌**"（板块整体走弱 → 系统性、个股非主因）的确定性依据，**比对大盘更贴近个股**。板块择时（板块价 vs MA20/MA60）随板块 K线一并给出。
- **同属热门板块（概念/题材，确定性）**：`analyze.py` 还从个股所属的**全部板块**（slist 一次抓回，行业之外的概念/题材/地域都在内）里，按**主力净流入**排序列出最强的 1-2 个，附「个股 vs 该板块」当日 RS。A股题材驱动，个股当日异动常由某概念（机器人/AI/低空等）带动，比行业板块更能解释——但 slist 不分概念/地域，**性质由模型判断**（题材共振 vs 仅地域同属）。
- **大盘均线择时（确定性）**：`analyze.py` 对上证/沪深300 给出 价 vs MA20/MA60 多空排列结论（偏多/偏空/震荡），替掉"A股市场情绪"那条 WebSearch——大盘背景直接采用脚本判断。
- **市场宽度 / 情绪温度（确定性，全市场）**：`analyze.py`（附带大盘指数时）一并给出**两市成交额**（流动性/风险偏好）、**涨停/跌停家数**（打板情绪 vs 退潮恐慌，比"涨跌家数"信号更强）、上证/深成/创业板涨跌幅。脚本据涨停/跌停直接给一句"情绪温度"分档（🔥偏暖/❄️恐慌/🌡️中性）。**这是情绪维的大盘背景锚，替掉"A股市场情绪/涨跌停"那条 WebSearch**。⚠️ 不含北向（个股与聚合实时数据均已停披）。
- 以下用 WebSearch：
  ```
  [股票名称] 今日 快讯 [当前日期]              # 公告走 news.py, 这里只补即时快讯/传闻
  [股票名称] 股吧 热帖 今日 [当前日期]
  [股票名称] 分析师 评级 研报 [当前日期]
  ```
  （大盘风险偏好/**两市成交额/涨跌停打板情绪**、**板块涨跌/板块主力净流入/个股 vs 板块 RS** 已由 `analyze.py` 给出，公告已由 `news.py` 给出，不再搜）

重点指标：换手率 vs 历史均值、分析师评级分布、社交热度趋势、大盘风险偏好、近期停板/连板。

**情绪温度计**：
- 🔥 过热：换手率 >历史均值2倍、研报清一色买入、社交热度暴增 → 警惕回调
- ❄️ 冰冷：换手率极低、无机构覆盖、舆情沉寂 → 可能左侧机会
- 🌡️ 适中：换手率正常、评级分化、讨论理性 → 相对健康

### 维度三：国际形势分析

**目标**：识别影响该股所在行业/公司的全球宏观因素。

并行搜索（含日期占位符）：
```
[股票所属行业] 国际 贸易 政策 [当前日期]
美联储 利率 政策 声明 [当前日期]
中美关系 贸易摩擦 最新动态 [当前日期]
[相关大宗商品/原材料] 国际价格 今日 [当前日期]
人民币汇率 今日 中间价 [当前日期]
```

> 按行业选词：出口型→汇率/贸易政策/目标市场需求；资源能源→大宗商品价格（原油/铜/铁矿石/煤炭）；科技半导体→出口管制/国产替代/全球芯片；消费医药→跨境监管/国际竞品/原料进口成本。

重点：贸易政策变化（关税/制裁/出口限制）、美联储走向、中美/中欧关系、大宗商品价格、海外同业动态。

### 维度四：经济形势分析

**目标**：评估宏观经济和行业景气度对该股基本面的支撑或压制。

- **估值水平**：PE-TTM/PB/市销率/股息率/市值的**当前值**，以及 PE/PB/PS **及股息率**的**近3/5年历史分位+护栏建议**，全部用 `scripts/valuation.py`（接口权威口径，勿搜摘要）。估值分位至此**全部由脚本直算，无需再 WebSearch**（股息率分位方向与估值相反，分位越高=越便宜）。

## Step 3：综合研判

四维信号交叉验证后量化加权，分三步走：先选行业权重模板 → 再按 rubric 给每维打分 → 最后算综合分映射方向。

> **算术交给 `scripts/score.py`，别手算。** 模型只负责 3.1 选模板、3.2 给每维「方向+强度」的**判断**；综合分、不计分维度的权重重新归一化、方向映射、估值护栏±1档、ST 封顶等**算术与映射**一律由脚本做（确定性、不漂口径、不出错）。`analyze.py` 已预填好命令，填入各维分即可：
>
> ```bash
> python3 scripts/score.py --template 消费医药 --val-pctl 1.5 \
>     --funds -3 --sentiment 0 --intl -2 --econ +2     # 看多+/看空-/中性0/不计分na, 强度即绝对值(|1|自动转不计分)
> # 选项: --val-pctl 估值主分位(valuation 给) --major-risk 重大单点风险 --st ST/*ST封顶 --weights 自定义权重
> ```
>
> 脚本直接输出综合分、最终方向、各项调档，以及可粘进报告的信号矩阵表。下面 3.1–3.3 是它内部规则的说明（便于核对），不必再手动复刻其算术。

### 3.1 选行业权重模板

**先看 `quote.py` 输出的「所属行业」**（东财行业，如〔白酒Ⅱ〕〔半导体〕〔股份制银行〕，已随行情一并返回，不必再猜或搜），把它归入下表某一类后选对应行（**每行合计 100%**），避免"固定权重"与"行业差异"自相矛盾。东财细分行业 → 模板大类的归类示例：白酒/食品/医药→消费/医药；半导体/光伏/电池/军工→科技成长；钢铁/煤炭/有色/石化→周期；银行/券商/保险→金融；其余不确定按默认/均衡。一只股票横跨多行业时取主营所属，或在两套模板间插值并说明理由。

| 行业类型 | 资金 | 情绪 | 国际 | 经济 | 侧重逻辑 |
|---------|----|----|----|----|---------|
| **默认/均衡**（不确定时用） | 30% | 20% | 20% | 30% | 四维均衡 |
| 周期股（钢铁/煤炭/石化/有色） | 25% | 15% | 30% | 30% | 大宗商品价格+行业景气主导 |
| 科技成长（半导体/新能源/AI/军工） | 25% | 20% | 30% | 25% | 出口管制+产业政策主导 |
| 消费/医药 | 25% | 20% | 15% | 40% | 国内经济/居民收入主导 |
| 金融（银行/券商/保险） | 30% | 15% | 15% | 40% | 货币政策+宏观流动性主导 |

### 3.2 给每维打分（方向 × 强度）

每维定**方向**（看多/利多 = `+`，看空/利空 = `−`，中性 = `0`）和**强度 1-5**，相乘得带符号分 `signed = 方向符号 × 强度`（中性恒为 0），范围 −5~+5。

强度 rubric（避免不同次分析口径漂移）：

| 强度 | 锚点 |
|----|------|
| 5 | 信号强烈且 ≥2 独立来源一致、数据当日已核验（如主力连续大幅净流入＋融资余额同步回升＋龙虎榜机构净买） |
| 4 | 信号明确、方向清晰，主要数据支持，无明显反向证据 |
| 3 | 信号中性或多空混杂、数据一般，方向不明显 |
| 2 | 信号偏弱，或关键数据仅单一来源/时效偏旧 |
| 1 | 信号微弱或当日数据缺失，几乎无指示意义 → **该维标"不计分"** |

> **数据不足的维度不硬凑分**：标"不计分"，把它的权重按比例分摊给其余维度（重新归一化到 100%）后再加权，并在研判中说明降权。不得用超窗口旧值凑强度（见【执行注意事项】）。
>
> **前置体检命中时**（Step 1.5）：停牌→相关维度不计分；ST/\*ST→看多侧强度封顶；次新→技术/估值相关信号不计分。

### 3.3 信号矩阵与综合分

| 维度 | 当前信号 | 方向(+/−/0) | 强度1-5 | 带符号分 | 权重 | 加权分 |
|------|---------|-----------|--------|--------|----|------|
| 资金流量 | [关键发现] | | | | %| |
| 市场情绪 | [关键发现] | | | | %| |
| 国际形势 | [关键发现] | | | | %| |
| 经济形势 | [关键发现] | | | | %| |

**综合分** = Σ(权重 × 带符号分)，范围 −5~+5，按下表映射整体方向：

| 综合分 | 整体方向 |
|------|---------|
| ≥ +3.0 | 强烈看多 |
| +1.5 ~ +3.0 | 偏多 |
| +0.5 ~ +1.5 | 中性偏多 |
| −0.5 ~ +0.5 | 中性 |
| −1.5 ~ −0.5 | 中性偏空 |
| −3.0 ~ −1.5 | 偏空 |
| ≤ −3.0 | 强烈看空 |

> 综合分是研判锚点而非唯一裁决：若存在重大单点风险（如停牌、业绩暴雷、监管处罚），可在结论中下调一档并说明。
>
> **估值护栏**：PE/PB 处于近3-5年**极高分位（>90%）→ 综合分下调一档**；**极低分位（<10%）且基本面无恶化 → 可上调一档**（安全边际）。分位与护栏建议**直接采用 `valuation.py` 输出**（接口直算，已给 `估值护栏[...]` 一行）；脚本因故未给出分位时才**不调档**、只作背景陈述；亏损股看 PB 分位/市销率，不以 PE 论。

## Step 5：研判留痕与复盘（闭环，可选）

为让研判**可追溯、可复盘**（也便于回头校准强度 rubric），输出报告后可把结论落痕，日后复盘命中率：

```bash
# 留痕: 把本次结论记一行(自动抓研判时现价作复盘基准; 综合分/方向用 score.py 结果)
python3 scripts/record.py --code 600519 --name 贵州茅台 --direction 偏多 --score 1.8 \
    --template 消费医药 --entry "1250-1280" --stop 1180 --target 1380 --note "主力回流+估值低位"

# 复盘: 对历史预测拉当前现价, 算前向收益与方向命中率
python3 scripts/review.py                 # 复盘全部
python3 scripts/review.py --code 600519    # 只看某标的
python3 scripts/review.py --min-days 5     # 只复盘记录满5天的(太新没走出来)
```

- `direction` 用 Step 3 的 7 档之一；`record.py` 自动抓研判时现价作复盘基准（抓不到可 `--ref-price` 补），**并同时抓基准指数点位（默认沪深300）供超额收益复盘**（`--bench` 改基准 / `--bench-ref-price` 手填 / `--no-fetch` 一并跳过）。
- `review.py` 判命中：看多且收益>带宽 / 看空且收益<−带宽 / 中性且 |收益|≤带宽（`--band` 默认 3%）；**同时给"超额收益（个股 − 基准）"口径的命中率**——大盘普涨时个股小涨在原始口径会误判"看多命中"，超额口径已剔除大盘 β、更反映研判本身的 α，两口径并列输出（老记录无基准点位时仅原始口径）。
- 命中率仅供**校准强度 rubric、发现系统性偏差**（如总体偏多但命中率低、或原始命中但超额常不中=只赚β不赚α）；样本少时不具统计意义。
- 预测日志默认写 skill 根目录 `predictions.jsonl`（可 `--file` 或环境变量 `ASHARE_PRED_LOG` 改）。

## 执行注意事项

**时效性窗口与超时处理**：

| 数据类型 | 可接受窗口 | 超时处理 |
|---------|--------------|---------|
| 盘中资金流向、实时行情 | 当日 | 标注"⚠️ 数据截至 [日期]，请核实" |
| 市场宽度（成交额/涨跌停家数，breadth.py） | 当日（盘中滚动） | 盘中为滚动值非收盘；不含北向 |
| 融资余额、龙虎榜 | 最近1个交易日 | 标注滞后并说明 |
| 技术位（均线/前高前低/换手分位） | 当日收盘 K线 | 脚本缺数据时价位留空，不臆造 |
| 分析师评级、研报 | 最近5个交易日 | 注明日期，不作主要信号 |
| 公告/事件（news.py） | 最近1-2周（按需 `--days`） | 标题+日期为硬事实；已发生事件作背景，不计入当日打分 |
| 未来事件（events.py：解禁/除权/业绩预告） | 未来日程（解禁不限，除权登记30天内提示） | 日期+数量+比例为硬事实；解禁=供给压力计入资金维，预告=基本面方向计入经济维，业绩预告超120天作背景 |
| 宏观/政策数据 | 最近7天 | 注明，降低权重 |
| 机构季报持仓、陆股通持股 | 最新一期 | 正常引用，标明报告期 |

> **个股北向/陆股通实时数据已于 2024-08-18 停披**：禁止搜"今日北向增减持 XX 万股"当作信号；只引用季报口径的陆股通持股（背景项）。详见【维度一】。

**滞后数据不进权重（重要）**：超窗口的数据（上月融资余额、上一交易日外的龙虎榜、几周前评级、季报口径的陆股通持股）只能作背景陈述，**不得计入信号矩阵的方向判断与打分**。

**脚本清单**：数据/评分脚本与用途见【数据获取】脚本总览表；每个脚本的字段口径、CLI 变体、运维工具（`selftest.py` 接口契约自检、`tests/` 离线单测、`--cache` 短TTL缓存）详见 **`reference/data-scripts.md`**。底层依赖：`trading_calendar.py`（沪深节假日表，被 quote 复用）。手抓接口表见 `reference/manual-data-interfaces.md`。

> 排查口诀：报告数字反常/某维总空/怀疑东财改字段 → 先跑 `python3 scripts/selftest.py`（FAIL=接口真变、THROTTLED=仅限流）；改确定性脚本后跑 `python3 -m unittest discover -s tests`。

1. `scripts/analyze.py` 编排器 / `quote.py`·`funds.py`·`valuation.py`·`news.py`·`events.py`·`breadth.py` 子脚本 / 实时 JSON 接口（最高，见【数据获取】）

6. 社交媒体情绪（仅辅助）

**行业差异化加权**：按行业选不同的四维权重模板，具体见 **Step 3.1 行业权重表**（不在此重复）。
